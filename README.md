### Задание 2

```
apiVersion: apps/v1
kind: Deployment
metadata:
  name: redis
spec:
  selector:
    matchLabels:
      app: redis
  replicas: 1
  template:
    metadata:
      labels:
        app: redis
    spec:
      containers:
      - name: master
        image: bitnami/redis
        env:
         - name: REDIS_PASSWORD
           value: password123
        ports:
        - containerPort: 6379
```


**Выполните действия:**

1. Измените файл с учётом условий:

 * redis должен запускаться без пароля;

 * версия образа redis должна быть зафиксирована на 6.0.13.

2. Запустите Deployment в своём кластере и добейтесь его стабильной работы.

# Решение

В рамках данной работы были развернуты и настроены базовые компоненты Kubernetes (Redis и Nginx) с использованием инструментов отладки и конфигурации.

## Выполненные задачи

### 1. Развертывание Redis (Задания 1-3)
* **Конфигурация**: Создан Deployment с использованием легковесного образа `redis:6.0.13-alpine`.
* **Сервис**: Настроен Service для обеспечения доступа к Redis внутри кластера.
* **Отладка**:
    * Проверена работа процессов внутри контейнера командой `ps aux`.
    * Проанализированы логи контейнера (`kubectl logs`), подтвержден статус `Ready to accept connections`.
    * Настроен проброс портов (`port-forward`) для проверки доступности Redis с локальной машины.

### 2. Развертывание Nginx с ConfigMap (Задание 4)
* **ConfigMap**: Создан объект конфигурации, содержащий настройки виртуального хоста для Nginx.
* **Deployment**: Развернут Nginx, в который через `volumeMounts` и `subPath` проброшен файл конфигурации из ConfigMap.
* **Связка ресурсов**: Настроены `Service` и `Ingress` для обеспечения внешнего доступа.
* **Проверка**: Веб-сервер успешно отдает кастомный ответ по пути `/test`.

## Основные команды управления

### Работа с Redis
```bash
# Просмотр процессов в поде
kubectl exec -it deployment/redis -- ps aux

# Просмотр логов (за все время или последние 5 минут)
kubectl logs deployment/redis --since=5m

# Проброс порта 6379 на локальный адрес
kubectl port-forward deployment/redis 6379:6379 --address 0.0.0.0
